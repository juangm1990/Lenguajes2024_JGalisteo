Resumen Tema 10: XML


¿Qué es XML (Extensible Markup Language)? Es un lenguaje de marcado de propósito general definido por el W3C. El propósito principal del lenguaje es compartir datos a través de diferentes sistemas, como Internet. Es similar a HTML y no está predefinido: "Las etiquetas se definen y se crean".

Introducción

XHTML es una versión más estricta de HTML que sigue las reglas de XML. Aunque fue importante en la evolución de los estándares web, en la práctica se utiliza HTML5, que es prácticamente igual a HTML.

  • Se trata de un estándar desarrollado por W3C. Es aplicar XML a HTML.
  • Versión más estricta y modular de HTML. Lo que significa que las etiquetas deben cerrarse correctamente y los documentos deben estar bien formados.
  • Sigue las reglas de XML para definir la estructura y el contenido de las páginas web.
  • Compatible con HTML.
  • Compatible con CSS.
  • El mismo uso que HTML.
  • Su extensión es .xhtml.

XML Extensible Markup Language

Características de XML:

  • Lenguaje de estructura libre. No está predefinido. Las etiquetas se definen.
  • Consta de dos partes: Prólogo y Cuerpo.
  • Usa espacios de nombres para evitar ambigüedades.
  • Su sintaxis es como la de HTML.
  • Debe cumplir las reglas W3C para estar bien formado.
  • Utiliza esquemas en forma de árbol con una única raíz.
  • Los documentos se validan.

Aclaraciones clave sobre XML:

  • No es un sustituto de HTML (ni lo pretende).
  • No es un gestor de base de datos.
  • No es un estándar de comunicación.
  • No es un lenguaje de programación.

Documentos bien formados

Cumple las reglas sintácticas definidas por el estándar XML de W3C.

Normas básicas:

  • Prólogo definido. Versión y codificación.
  • Estructura de árbol. Un único nodo raíz.
  • Etiquetas emparejadas. Apertura y cierre.
  • Atributos entre comillas. Simples o dobles.
  • Distingue mayúsculas de minúsculas.
  • Nombres de etiquetas. Similares a variables en programación.
  • Comentarios fuera de etiquetas con: <!-- comentario -->.

Validación con DTD

Un DTD describe la estructura del documento:

  • Elementos
  • Orden

Ubicación de DTD:

  • Interno. En el propio documento
  • Externo. Referenciado por URL
  • Mixto

Tipos de elementos que se utilizan en DTD:

  • Elemento <!ELEMENT>
  • Atributos <!ATTLIST>
  • Entidades <!ENTITY>

1. Elemento <!ELEMENT>

  • <!ELEMENT elemento (hijo1, hijo2 ..)>
  • <!ELEMENT elemento (#PCDATA)>

Cardinalidad:

  • ?: El elemento aparece 0 o 1 vez.
  • *: El elemento aparece 0 a N veces.
  • +: El elemento aparece 1 a N veces.

Secuencia:

  • X, Y: Indica que el elemento X tiene que aparecer a continuación de Y.
  • (X|Y): Indica que aparece el elemento X o el elemento Y, pero no los dos al mismo tiempo.

Combinación:

  • Se pueden combinar los símbolos de secuencia y los de cardinalidad.

2. Atributos <!ATTLIST>

Información adicional de los elementos:

<!ATTLIST elemento atributo tipo valor>.
Ejemplo: <!ATTLIST capitulo paginas CDATA #REQUIRED>

Tipos

Propiedad Significado
CDATA Cadena de texto. Valor alfanumérico.
Enumeraciones Permite indicar varios valores.
ENTITY Entidad definida previamente.
ENTITIES Conjunto de entidades.
IDREF Se referencia el identificador de otro elemento.
IDREFS Referencia a un conjunto de identificadores.
ID Identificador único para identificar elementos.
NOTATION Indica datos que no son XML.
NMToken Un nombre sin espacios en blanco.
NMtokens Lista de nombres.

Valores:

Propiedad Significado
Valor El valor por defecto del atributo entre comillas.
#REQUIRED El atributo es obligatorio.
#IMPLIED El atributo es opcional.
#FIXED valor El atributo es obligatorio y se le asigna un valor por defecto que es el único valor que puede tener.

Puntos clave que no se pueden ignorar

¿Qué es el lenguaje XML? Usos y características:

XML es un lenguaje de marcado utilizado para almacenar y transportar datos de manera estructurada. Es flexible, legible para humanos y máquinas, y se usa para el intercambio de datos entre sistemas.

Estructura de un documento XML y sintaxis de un elemento XML:

Un documento XML tiene un prologo, un elemento raíz y elementos con atributos. Ejemplos:
<libro titulo="XML básico">
<autor>Juan Pérez</autor>
<anio>2023</anio>
</libro>

¿Qué es la validación de un documento XML? ¿Qué formas hay?

La validación asegura que el XML cumple con las reglas definidas. Se puede hacer por DTD o XML Schema.

Validación DTD:

La validación DTD verifica que el documento cumpla con las reglas definidas en un archivo DTD.
