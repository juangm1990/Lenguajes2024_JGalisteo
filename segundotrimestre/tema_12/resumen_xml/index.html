Resumen Tema 12: Conversión y adaptación de documentos XML.


XML es un formato de datos estructurado utilizado para almacenar y compartir información de manera jerárquica. Para modificar y transformar documentos XML, se emplean lenguajes como XSLT, que permite convertirlos en otros formatos, y XPath, que facilita la selección de elementos dentro del árbol XML. Además, existen otros formatos de intercambio de datos como JSON, CSV y YAML, cada uno con características específicas según su uso.

Lenguajes de Transformación XSL

XSL (eXtended Stylesheet Language) es un conjunto de tecnologías para transformar y dar formato a documentos XML. Su uso permite manipular la estructura del contenido de manera similar a cómo CSS estiliza documentos HTML.

  • XPath: Permite evaluar expresiones y localizar elementos dentro de un documento XML.
  • XSLT: Facilita la transformación de documentos XML en otros formatos, como HTML o XML con diferente estructura.
  • XSL-FO: Se emplea para generar documentos en otros formatos, como PDF, aplicando reglas de formato avanzadas.

¿Para qué sirve XPath?

XPath es un lenguaje que permite navegar y seleccionar nodos dentro de un documento XML. Es ampliamente utilizado en procesos de transformación y extracción de datos.

  • Acceso a los nodos: Permite identificar y recorrer los elementos de un XML.
  • Extracción de datos: Facilita la recuperación de información específica dentro del documento.
  • Visualización de elementos: Ayuda a estructurar y presentar el contenido de un XML.
  • Hacer cálculos: Posibilita realizar operaciones sobre los valores extraídos.

Asociación del esquema (XSD) al archivo XML

<?xml version="1.0" encoding="UTF-8"?>

<concesionario xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

xsi:noNamespaceSchemaLocation="concesionario.xsd">

  <coche precio="10000">

    <marca></marca>

    <modelo></modelo>

    <matricula></matricula>

    <combustible></combustible>

  </coche>

</concesionario>

<?xml version="1.0" encoding="UTF-8"?>

<?xml-model href="concesionario.xsd"?>

<concesionario>

  <coche precio="10000">

    <marca></marca>

    <modelo></modelo>

    <matricula></matricula>

    <combustible></combustible>

  </coche>

</concesionario>

Estructura de un esquema XML (XSD)

<?xml version="1.0" encoding="UTF-8"?>

<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">

<xs:element name="concesionario">

<xs:complexType>

<xs:sequence>

<xs:element name="coche">

<xs:complexType>

<xs:sequence>

<xs:element name="marca" />

<xs:element name="modelo" />

<xs:element ref="matricula" />

<xs:element name="combustible" />

</xs:sequence>

<xs:attribute name="precio" type="xs:integer" use="required" />

</xs:complexType>

</xs:element>

</xs:sequence>

</xs:complexType>

</xs:element>

<xs:element name="matricula" type="xs:string"/>

</xs:schema>

Definición de elementos en XML Schema (XSD)

Elementos Simples

Los elementos simples en XML Schema contienen datos pero no otros elementos o atributos.

Ejemplo:

<xs:element name=”ejemplo” type=”tipo” minOccurs= “valor”
maxOccurs = “valor” fixed = ”valor” default = ”valor”/>

Elemento personalizado-Complejos

Los elementos complejos pueden contener otros elementos y atributos, permitiendo estructuras más elaboradas.

Ejemplo:

<xs:element name="ejemplo">
  <xs:complexType>
    <xs:sequence> … </xs:sequence>
  </xs:complexType>
</xs:element>

Elemento personalizado-simples

Los elementos personalizados simples permiten aplicar restricciones sobre el tipo de dato definido.

Ejemplo:

<xs:element name="ejemplo">
  <xs:simpleType>
    <xs:restriction> … </xs:restriction>
  </xs:simpleType>
</xs:element>

Tipos de datos para elemento personalizado-Complejos

Los elementos personalizados complejos permiten definir estructuras de datos más avanzadas en XML Schema.

Se pueden usar diferentes modelos de contenido como sequence, choice y all para organizar los elementos dentro de una estructura compleja.

Ejemplo de elemento complejo con xs:sequence y xs:choice

En este ejemplo, el elemento ejemplo contiene una secuencia de elementos, donde hay una elección obligatoria entre dos opciones (precio y precioiva).

Ejemplo de código XML Schema:

<xs:element name="ejemplo">
  <xs:complexType>
    <xs:sequence>
      —--Elementos—-
      <xs:choice>
        —elementos a elegir– (ej: precio y precioiva)
      </xs:choice>
      —--más elementos—-
    </xs:sequence>
  </xs:complexType>
</xs:element>

Enumerados y Listas en XML Schema (XSD)

En XML Schema, los tipos de datos personalizados simples permiten definir restricciones sobre los valores de un elemento.

Enumerados (Personalizados simples)

Los enumerados restringen un elemento a un conjunto específico de valores permitidos.

Ejemplo de código:

<xs:element name="color">
  <xs:simpleType>
    <xs:restriction base="xs:string">
      <xs:enumeration value="rojo"/>
      <xs:enumeration value="verde"/>
      <xs:enumeration value="azul"/>
    </xs:restriction>
  </xs:simpleType>
</xs:element>

Listas (Personalizados simples)

Las listas permiten definir un elemento que puede contener múltiples valores separados por espacios.

Ejemplo de código:

<xs:element name="colores">
  <xs:simpleType>
    <xs:list itemType="xs:string"/>
  </xs:simpleType>
</xs:element>

Atributos en XML Schema (XSD)

Los atributos en XML Schema se utilizan para proporcionar información adicional sobre los elementos.

Se definen dentro de los elementos y pueden tener restricciones, valores predeterminados o ser obligatorios.

Ejemplo de Atributo Básico

Definición de un atributo con nombre, tipo y reglas de uso.

Ejemplo de código:

<xs:attribute name=”nombre” type=”tipo” use= “valor”
default= “valor” fixed = ”valor” />

Ejemplo de Atributo sin Tipo Específico

Un atributo sin restricciones adicionales.

Ejemplo de código:

<xs:attribute name="editorial" />

Ejemplo de Atributo con Restricciones (Enumerados)

Un atributo que solo acepta ciertos valores predefinidos (baja, media, alta).

Ejemplo de código:

<xs:attribute name="demanda">
  <xs:simpleType>
    <xs:restriction base="xs:string">
      <xs:enumeration value="baja"/>
      <xs:enumeration value="media"/>
      <xs:enumeration value="alta"/>
    </xs:restriction>
  </xs:simpleType>
</xs:attribute>
